1.	Purpose of the App

This application's goal is to assist users in differentiating between false internet myths and practical productivity tips. Many people heed popular advice without checking its veracity, which can occasionally be detrimental or even ineffective. Through a flashcard-style quiz, this software offers users an interactive and entertaining approach to assess their knowledge and learn accurate information.

2. The app’s objectives

The primary goals are:
•	To educate users about common life hacks and myths
•	To provide an interactive quiz using flashcards
•	To improve user decision-making and critical thinking
•	To track user performance through a scoring system
•	To give instant feedback for each answer
•	To allow users to review correct answers after completing the quiz

3. Design Considerations

The app was designed with the following considerations:
•	Simplicity: The interface is clean and easy to understand for all users
•	Consistency: Colours, buttons, and layouts are consistent across all screens
•	Responsiveness: The app adapts well to different screen sizes
•	Readability: Text sizes and colours are chosen for easy reading
•	User Engagement: Bright colours and interactive buttons keep users interested

3.1 Design of User Interfaces

The app consists of three main screens:

1. Welcome Screen
•	Displays the app title and description
•	Includes a “Start Quiz” button
•	Uses a colourful header and centered layout

2. Quiz Screen
•	Displays one question at a time
•	Two buttons: Hack (True) and Myth (False)
•	Feedback text appears after answering
•	“Next” button to move through questions

3. Score Screen
•	Displays total score
•	Provides personalized feedback
•	Includes a “Review Answers” button
•	Shows explanations for all questions


3.2 User Experience (UX)

The application is made to offer seamless and pleasurable user experience:
•	Simple screen switching
•	Instant feedback following each response
•	Labels and instructions that are clear
•	Quick and uninterrupted performance
•	Using colours and iconography in an eye-catching design
•	Logical progression: Quiz → Start → Score → Review


4. Handling Errors


To ensure the app runs smoothly, the following error handling techniques were implemented:
•	Prevents crashing when moving between questions
•	Ensures the app does not go beyond the last question
•	Validates user interaction before proceeding
•	Uses Kotlin safe coding practices
•	Logging (Log.d) is used to debug issues
•	Default values are provided where necessary


5. Testing

•	The app was tested on an Android emulator
•	All buttons and navigation were tested
•	Score calculation was verified
•	No crashes occurred during testing


6. Technologies used

•	Kotlin
•	Android Studio
